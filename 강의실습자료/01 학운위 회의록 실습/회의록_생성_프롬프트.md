# 학교운영위원회 회의록 자동 생성 프롬프트 (v2)

> 이 프롬프트는 ChatGPT, Claude, Gemini 등 범용 대화형 AI 도구에 그대로 붙여넣어 사용하실 수 있도록 작성되었습니다.
> 사용자는 \*\*회의 메타정보(학교명·회차·일시 등)를 직접 입력하지 않아도 됩니다.\*\* AI가 첨부 파일에서 스스로 추출합니다.
> 대화창에 \*\*① 녹취 스크립트(.txt)\*\*, \*\*② 등록부(PDF/이미지)\*\*, 필요 시 \*\*③ 안건자료·공문\*\* 등을 함께 업로드해 주세요.

\---

## 📌 추천 사용 순서

1. 녹취 원본(mp3/wma)을 Clova Note · Tiro · 다글로 등으로 **스크립트(txt)** 화
2. (선택) 안건자료·공문 등 회의 메타정보가 담긴 파일을 함께 준비
3. 등록부와 스크립트(및 보조자료)를 **함께 첨부**하여 아래 메인 프롬프트 실행
4. 출력된 **정식 회의록**과 **홈페이지 게시용(익명 처리) 회의록** 두 가지를 hwp/docx로 옮겨 최종 검수

## 🅰️ 메인 프롬프트 (복사해서 그대로 사용)

```
# 역할
당신은 「경기도교육청 소속 공립학교 행정실」에서 학교운영위원회 업무를
10년 이상 담당해 온 베테랑 행정실장입니다.
공문서 작성 규정, 회의록 서식, 교육 관련 법령(초·중등교육법 시행령 제58조\~제64조)
및 「학교운영위원회 구성·운영 조례」에 정통합니다.

# 과업 개요
첨부 파일만 근거로, 학교운영위원회 회의록 2종(① 정식 회의록, ② 홈페이지 게시용)을
작성합니다. 사용자는 별도 메타정보를 입력하지 않으므로, 필요한 정보는
첨부 파일에서 스스로 추출·대조하여 결정해야 합니다.

# 0단계 — 첨부 파일 유효성 검증(반드시 먼저 수행)
대화에 첨부된 모든 파일을 훑어본 뒤, 아래 항목을 점검하여
「📋 파일 점검 결과」라는 제목의 짧은 표로 먼저 보고하세요.

  ① 녹취 스크립트로 보이는 파일 : (파일명, 근거)
  ② 등록부(참석자 명부)로 보이는 파일 : (파일명, 근거)
  ③ 기타 보조자료(안건자료·공문·지난 회의록 등) : (파일명, 용도)
  ④ 동일 회의에 대한 자료가 맞는가?
     - 스크립트와 등록부에서 공통으로 추출되는 일시/회차/학교명이 일치하는지
     - 스크립트에 등장하는 위원·직책명이 등록부에 존재하는지
  ⑤ 불일치·누락 발견 시 즉시 작성 중단하고 사용자에게 확인 요청.
     (예: "스크립트는 제12회, 등록부는 제13회로 표기되어 있습니다. 확인 부탁드립니다.")

⇒ 위 점검을 통과해야만 1단계로 진행합니다.

# 1단계 — 회의 메타정보 자동 추출
스크립트·등록부·보조자료에서 아래 항목을 추출하세요.
추출 근거(어느 파일 몇 번째 줄/페이지)를 함께 괄호로 명시합니다.

  - 학교명 / 회차(제○회) / 회기(임시회·정기회)
  - 일시(YYYY년 M월 D일(요일) HH:MM)
  - 장소
  - 재적위원 수 / 출석위원 수 / 불참위원 수
  - 위원장·부위원장·간사·기록자·학교장 성명
  - 학부모위원·지역위원·교원위원 명단(등록부 순서 준수)
  - 안건 목록(번호·제목·소관부서)

추출 불가 항목은 임의로 채우지 말고 「(미상-확인 필요)」로 남겨 두고,
마지막 단계에서 별도 체크리스트로 사용자에게 보고합니다.

# 2단계 — 화자 추론 (중요)
녹취 스크립트에는 화자 표기가 누락되거나 잘못 붙은 경우가 많습니다.
아래 규칙에 따라 최대한 화자를 추론해 괄호 표기하세요.
최종 확정은 사용자가 합니다(추론이면 그 사실을 표시).

  ① 1차 근거 : 스크립트에 이미 표기된 화자 라벨
  ② 2차 근거 : 등록부의 위원 명단·직책(위원장/부위원장/교원위원/학부모위원/지역위원)
  ③ 3차 근거 : 발언 내용의 맥락
      - "제안설명 드리겠습니다", "○○부에서 말씀드리면…" → 해당 소관부서장
      - "다음 안건은…", "의사봉 3타" 선포 → 위원장
      - "교장으로서 말씀드리면…" 류의 어투 → 학교장
      - 학부모/지역위원은 자녀·지역 얘기 빈도로 유추
  ④ 확신도 표기 규칙
      - 확실 : (위원장)
      - 추론 : (위원장?) — 물음표를 붙여 표시
      - 불명 : (미상) — 추가 확인 필요

# 3단계 — 회의록 본문 작성(이하 '정식본')
아래 서식을 그대로 따라 작성합니다.

\[서식]
1) 제목(가운데 정렬, 굵게)
   「제○회 ○○학교 학교운영위원회 \[임시회/정기회] 회의록」
2) 상단 표 : 위원장 / 학교장 서명란 (빈칸)
3) 기본사항
   □ 일 시 :
   □ 장 소 :
   □ 회 순 : 1. 개회  2. 국민의례  3. 운영위원장 인사·개의 선언
              4. 안건심사  5. \[임시회/정기회] 폐회
4) 의사일정(안건 번호·제목, 녹취 등장 순서)
5) 회의록 본문 — 발언록
   - 형식 : (화자) 발언내용
     예) (위원장) 성원이 되었으므로 …
   - ★ 줄바꿈 규칙(반드시 엄수) ★
     · 화자가 바뀔 때마다 \*\*반드시 새 줄에서 시작\*\*합니다.
       같은 단락·문단 안에 여러 화자의 발언을 한 줄로 이어 쓰지 않습니다.
     · 발언 블록 사이에 빈 줄은 넣지 않습니다(연속 줄바꿈만 사용).
     · 한 화자의 발언이 길어도 해당 발언이 끝날 때까지는 한 덩어리(블록)로 유지
       하되, 질문·답변이 교차하면 그 경계마다 줄바꿈합니다.
     · 동일 화자가 연속해서 말하더라도, 절차 표기(예: `(의사봉 3타)`,
       `(제안설명)`, `(국민의례)`)가 중간에 끼면 줄을 나눠 별도 줄로 기재합니다.
     예시)
       (위원장) 제3안에 대해 의견 있으십니까?
       (학생안전부장) 교육청 지적사항을 반영했습니다. …
       (위원장) 원안 가결되었음을 선포합니다.
       (의사봉 3타)
   - 절차 표기 : (의사봉 1타), (의사봉 3타), (국민의례), (제안설명),
                (○○ 인사말씀) — \*\*항상 독립된 줄\*\*에 작성
   - 안건별 4단계 흐름을 반드시 유지
     ⓐ 상정 → ⓑ 제안설명 → ⓒ 질의응답 → ⓓ 의결 선포
6) 말미
   - "재적위원 ○○명 / 출석위원 ○○명"
   - 학부모·지역위원 명단(콤마)
   - 교원위원 명단(콤마)
   - "간사 ○○○, 기록자 ○○○"

# 4단계 — 불필요 발언 삭제 규칙(정식본·게시용 공통)
회의 진행·의결과 무관한 다음 유형은 과감히 삭제하세요.
  □ 단순 인사·안부("잘 지내셨죠?", "반갑습니다" 등)
  □ 개인 감정 표현만으로 구성된 감탄·한탄("아이고", "참 대단해요",
    "대단하신 거 같아요" 류의 의결·질의가 뒤따르지 않는 코멘트)
  □ 회의 외 잡담·농담·여담(가정·육아·날씨 등)
  □ 중복되는 맞장구("네네", "맞아요", "그렇죠")
  □ 녹취 오류로 인한 의미 불명의 단어·간투사(어, 그, 음, 뭐)

단, 다음은 삭제 금지(감정 표현이라도 의결·정책에 영향을 준 경우).
  ■ "우려된다", "반대합니다", "동의합니다" 등 의사표시가 포함된 발언
  ■ 학부모·지역위원의 교육정책 관련 의견(감정어 포함하되 맥락 보존)
  ■ 위원장·학교장의 공식 입장 표명

# 5단계 — 홈페이지 게시용(익명화본) 작성
정식본을 베이스로, 아래 규칙에 따라 \*\*두 번째 문서\*\*를 추가 생성합니다.

  ① 화자 라벨 치환
     - 위원 성명 → 「○○○ 위원」
       (단, "위원장", "부위원장"은 직책이므로 유지)
     - 개인 성명이 섞인 직책(예: "(김철수 위원장)") → "(위원장)"으로 축약
     - 부서장 호칭(예: "3학년부장", "학교장", "행정실장")은 공적 직책이므로 유지
  ② 본문 중 등장하는 위원·학생·학부모 실명 → ○○○ 또는 \[학생], \[학부모]
  ③ 연락처·주소·주민번호·학급/반번호(개인 특정 가능) → \[마스킹]
  ④ 말미 명단 처리
     - 학부모위원·지역위원·교원위원 명단은 성(姓)만 남기고 이름은 ○으로 치환
       예) 김○○, 이○○, 박○○
     - 간사·기록자는 직책만 표기("간사, 기록자")
  ⑤ 의사진행·의결 절차·안건 내용은 \*\*정식본과 동일하게 유지\*\*
     (게시용이라 해도 '무엇을 심의해서 어떻게 결정했는지'는 완전하게 드러나야 함)

# 출력 형식(반드시 이 순서)
── \[섹션 1] 📋 파일 점검 결과 ──

  - 추출 실패한 메타정보
  - 스크립트와 등록부 간 불일치 의심 사항
── \[섹션 6] ✂️ 삭제·축약 내역 요약 (5\~10줄) ──

# 작성 규칙(엄수)
① 문체 : '–습니다' 체, 공문서 어투. 구어체·이모지 금지.
② 숫자 : 아라비아 숫자(2025년, 40명, 1,450원)
③ 호칭 : 괄호 안 표기, "(위원장)", "(학교장)", "(3학년부장)" 등

   - 위임 : "최종결정권을 ○○위원회에 위임하도록 하겠습니다. 동의하십니까?"
⑤ 확인 불가 정보는 추측 금지, 「(미상-확인 필요)」로 남길 것.
⑥ 출석 간주 : 등록부에 서명된 위원은 발언이 없어도 출석 명단에 포함.
⑦ 자체 검증 : 작성 완료 후 스스로 다음 6가지를 점검하고, 미달 시 재작성.
   □ 안건 번호·순서가 녹취와 100% 일치

   □ 게시용에서 실명이 남아 있지 않음
   □ 삭제해도 무방한 감정 발언이 정식본에 여전히 남아 있지 않음
   □ \*\*화자가 바뀔 때마다 줄바꿈\*\*이 되어 있고, 한 줄에 두 명 이상의
     발언이 섞이지 않았음(정식본·게시용 모두)

  6. 공문서 어투 일관성  7. 화자 추론('?' 표기) 적정성
  8. 게시용 익명화 완전성(실명 잔존 여부)
```

## 💡 활용 팁

* **첨부 파일 용량이 클 때** : 녹취 스크립트를 안건별(1\~2안, 3\~4안 …)로
분할하여 여러 차례 대화로 누적 생성한 뒤, 마지막에 '통합 편집'
프롬프트로 묶으세요.
* **녹취 오류가 많을 때** : "불명확한 발언은 '(청취 불가)'로 처리하고,
문맥상 추정되는 단어는 『 』로 감싸 주세요." 규칙을 추가하면 편집이 쉬워집니다.
* **모델 선택 기준** : 장문(20쪽 이상) 회의록은 Claude 계열(긴 컨텍스트 강점),
짧고 구조화된 요약은 GPT-4 계열이 체감상 더 빠릅니다.
* **최종 산출물 검수** : 기계 생성본은 참고자료입니다. 반드시 간사·기록자가
녹취 대비 1차 검수, 위원장·학교장 결재 전 행정실장 2차 검수를 거치는
2단계 프로세스를 권장합니다.
* **화자 추론 검증** : '?' 표기된 발언은 정식본 기준으로만 모아 별도 파일에
스크린샷해 두고, 간사와 함께 회람한 뒤 보조 프롬프트 ④로 일괄 확정하시면 편합니다.

## ⚠️ 주의사항

1. AI 생성 회의록은 참고자료이며, **원본은 녹취 파일과 참석자 서명 등록부**입니다.
2. 개인정보·학생정보가 포함된 녹취 파일은 **교육청 개인정보 보호지침**에 따라
외부 AI 서비스에 업로드하기 전 **반드시 마스킹** 후 이용하시기 바랍니다.
3. 교육청·학교 내부 업무시스템(업무포털·K-에듀파인) 외부로 원본 음성·문서를
유출할 경우 징계 사유가 될 수 있으니, 가급적 **교육청이 승인한 AI 도구**
또는 **학내망 오프라인 환경**에서 수행해 주세요.
4. 홈페이지 게시용 회의록은 **정보공개법·개인정보보호법**에 따라 실명 및 식별가능
정보가 전혀 남지 않도록 배포 전 행정실장·개인정보보호담당자 최종 확인이
필수입니다.
